# 公募基金风险收益评价与决策辅助系统项目报告

## 1. 项目背景

传统基金筛选如果只关注历史收益率，容易忽略波动、回撤和持有体验。因此，本项目尝试构建一个以 Python 为核心的基金风险收益评价系统，将基金筛选从单一收益排序扩展为多指标综合分析。

项目定位是：

> 基于真实基金净值数据的历史表现分析和决策辅助系统。

系统不构成个性化投资建议，不输出买入、持有或卖出指令。所有结论仅用于历史数据分析和研究比较。

## 2. 数据来源

系统使用 AkShare 获取开放式基金公开数据，包括：

- 基金代码
- 基金简称
- 单位净值走势

系统也支持用户导入自己的宽表 CSV 数据，格式为：

```csv
Date,Fund_A,Fund_B,Fund_C
2023-01-03,1.0000,1.0000,1.0000
2023-01-04,1.0021,0.9987,1.0032
```

Web 版会将抓取到的基金净值保存到本地 SQLite 数据库，后续重复分析同一基金时优先读取缓存，减少对外部接口的依赖。

## 3. 系统架构

```text
用户输入基金代码 / 选择基金池
        ↓
AkShare 真实数据抓取
        ↓
SQLite 本地缓存
        ↓
净值数据清洗与收益率计算
        ↓
风险收益指标计算
        ↓
多因子评分模型
        ↓
投资者画像排名与敏感性分析
        ↓
风险等级与决策辅助标签
        ↓
Web 展示 / CSV 下载 / Markdown 报告
```

核心模块：

| 模块 | 作用 |
|---|---|
| `akshare_data.py` | 真实基金数据抓取和基金搜索 |
| `storage.py` | SQLite 缓存、基金池、分析历史 |
| `metrics.py` | 收益率、波动率、最大回撤等指标计算 |
| `scoring.py` | 多因子评分和投资者画像 |
| `advisory.py` | 风险等级和决策辅助标签 |
| `pipeline.py` | 统一分析管线 |
| `web.py` | FastAPI Web 分析台 |
| `fund_universe.py` | 基金池准入、同类基金筛选和份额去重 |
| `diagnostics.py` | 因子相关性诊断 |
| `explainability.py` | 评分因子贡献分解 |
| `validation.py` | Walk-Forward 样本外验证 |
| `research.py` | 研究附录生成 |

## 4. 指标体系

系统计算以下指标：

| 指标 | 含义 |
|---|---|
| Annual Return | 年化收益率，衡量长期收益能力 |
| Annual Volatility | 年化波动率，衡量收益波动程度 |
| Maximum Drawdown | 最大回撤，衡量历史最大亏损幅度 |
| Sharpe Ratio | 单位波动承担下获得的超额收益 |
| Calmar Ratio | 年化收益相对于最大回撤的表现 |
| Rolling Positive Ratio | 60 日滚动收益为正的比例，衡量稳定性 |

这些指标覆盖了收益、风险、风险调整收益和稳定性四个维度。

## 5. 多因子评分模型

系统先将不同量纲的指标转换为百分位得分，再按投资者画像加权汇总：

```text
Fund Score =
w1 * Annual Return
+ w2 * Sharpe Ratio
+ w3 * Maximum Drawdown
+ w4 * Calmar Ratio
+ w5 * Annual Volatility
+ w6 * Rolling Positive Ratio
```

内置三类投资者画像：

| 画像 | 设计思路 |
|---|---|
| aggressive | 更看重收益和收益持续性，接受较高波动 |
| balanced | 平衡收益、回撤、风险调整收益和稳定性 |
| conservative | 更看重回撤控制、波动控制和持有体验 |

系统还会生成权重敏感性分析，比较基金在不同风险偏好下的排名变化。

此外，系统会对评分因子进行 1% 和 99% 分位数缩尾处理，降低极端指标值对综合评分的影响。

## 6. 风险等级与决策辅助标签

系统根据历史波动率、最大回撤、综合评分和数据充分性生成：

- 风险等级：较低风险、中等风险、中高风险、高风险
- 决策辅助标签：重点观察、可观察、高回撤预警、暂不优先
- 标签原因：解释为什么给出该标签

这些标签用于研究辅助，不构成投资建议。

## 7. Web 功能

Web 分析台支持：

- 输入基金代码并分析。
- 按基金名称或代码搜索基金。
- 选择默认基金池。
- 保存和删除自定义基金池。
- 查看排名结果、风险等级和标签原因。
- 查看基金类型、同类排名、数据质量提示和自然语言解释。
- 查看本地数据状态，包括最早日期、最新日期和数据条数。
- 打开历史分析结果。
- 复用历史参数重新分析。
- 下载 Markdown 报告、CSV 排名和指标明细。
- 查看研究报告。
- 查看基金池准入报告、因子相关性诊断、因子贡献解释、权重扰动稳健性分析和 Walk-Forward 样本外验证报告。

## 8. 模型验证与诊断

系统新增四类研究型输出：

- 基金池准入：按主动权益类可比原则进行筛选，并对 A/C 份额做代表份额保留。
- 因子相关性诊断：用 Spearman 相关系数识别 Sharpe、Calmar、收益率等指标之间的信息重叠。
- 因子贡献分解：由于评分模型是线性加权模型，可以直接计算每个因子对综合评分的精确贡献。
- Walk-Forward 样本外验证：使用滚动历史窗口评分，并观察后续持有期 Top 组合表现。

Walk-Forward 的结论边界是：它不证明模型可以预测未来收益，只检验历史评价指标是否对后续表现具有一定区分能力。

## 9. SQLite 数据库设计

系统使用本地 SQLite 数据库：

```text
data/fund_ranking.db
```

主要表：

| 表 | 内容 |
|---|---|
| `funds` | 基金代码、基金名称、更新时间 |
| `fund_nav` | 基金代码、净值日期、单位净值 |
| `analysis_runs` | 分析时间、基金组合、画像、报告路径 |
| `fund_pools` | 用户保存的基金池名称和基金代码 |

数据库的作用：

- 减少重复抓取。
- 保留分析历史。
- 支持基金池管理。
- 支持独立结果页。

## 10. 研究附录

当前研究附录包含：

- 基于基金名称关键词的轻量文本因子。
- 基于历史指标构造的代理目标 `future_score_proxy`。
- 标准化线性模型解释各因子影响方向和重要性。

需要强调的是，当前研究附录不是真实收益预测模型，也不等同于基于基金季报或公告构建的正式文本因子。

## 11. 项目局限性

主要局限包括：

- AkShare 数据源可能受接口变更、网络和限流影响。
- 当前评分权重仍带有主观假设。
- 历史表现不代表未来收益。
- 当前研究附录使用的是研究代理目标，不是真实未来收益标签。
- 系统没有用户风险测评和持牌投顾合规流程，因此不能提供个性化投资建议。

## 12. 总结

本项目完成了从真实基金数据抓取、指标计算、多因子评分、权重敏感性分析、Web 展示、SQLite 缓存、基金池管理、历史结果页到研究报告的完整闭环。

它适合作为金融数据分析和量化研究原型进行展示，重点体现：

- 金融业务理解
- Python 数据分析能力
- 工程化项目组织能力
- Web 应用开发能力
- 数据持久化和缓存设计能力
- 可解释分析意识
