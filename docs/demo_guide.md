# Web 演示说明

## 1. 启动系统

```bash
cd ~/fund-ranking-system
.venv/bin/fund-ranking-web --host 127.0.0.1 --port 8000
```

打开：

```bash
open http://127.0.0.1:8000
```

## 2. 推荐演示流程

### 第一步：说明项目目标

可以这样介绍：

> 这个系统用于公募基金历史表现分析和决策辅助。它不是简单按收益率排名，而是综合考虑收益、波动、最大回撤、Sharpe、Calmar 和稳定性，并支持不同风险偏好的投资者画像。

### 第二步：输入基金代码

在“基金代码”中输入：

```text
000001 000011 000083
```

起始日期使用：

```text
2021-01-01
```

投资者画像选择：

```text
balanced
```

点击“开始分析”。

### 第三步：解释加载和缓存

页面会显示：

```text
正在抓取真实基金净值并生成报告，请稍等几秒...
```

如果本地数据库已有数据，会显示：

```text
缓存命中 3 只，远程补抓 0 只。
```

这说明系统不是每次都重复请求数据源，而是优先复用本地 SQLite 缓存。

### 第四步：解释排名结果

重点看：

- 综合评分
- 风险等级
- 决策辅助标签
- 标签原因

演示话术：

> 系统会先计算每只基金的风险收益指标，再做百分位归一化和加权评分。排名结果旁边的风险等级和决策辅助标签不是买卖建议，而是用于缩小研究范围。

### 第五步：解释本地数据状态

“本地数据状态”会显示：

- 最早日期
- 最新日期
- 数据条数

演示话术：

> 这部分用于检查数据覆盖是否充分。如果某只基金数据条数太少，就不适合直接和长期历史基金比较。

### 第六步：打开历史结果

在“最近分析历史”中点击：

- `打开`：查看某次独立分析结果
- `复用`：把历史参数带回首页重新分析

演示话术：

> 每次分析都会生成独立结果页和独立报告目录，因此历史结果不会被覆盖。

### 第七步：展示基金池管理

保存一个基金池：

```text
名称：我的观察池
基金代码：000001 000011 000083
```

保存后可以从“默认基金池”下拉框中选择。

### 第八步：下载结果

“下载结果”中可以下载：

- Word 综合报告
- PDF 综合报告
- Excel 数据汇总
- 主分析报告
- 权重敏感性报告
- 机器学习辅助评分报告
- 研究报告
- 当前画像排名 CSV
- ML 辅助排名 CSV
- ML 学习权重 CSV
- 全部画像排名 CSV
- 指标明细 CSV

## 3. 功能展示重点

优先展示这 5 个点：

1. 真实基金数据接入。
2. 多因子评分模型。
3. 三类投资者画像、权重敏感性分析和机器学习辅助评分。
4. SQLite 缓存、历史记录和独立结果页。
5. 合规边界：系统是研究辅助，不是个性化投资建议。

## 4. 常见异常说明

如果基金代码抓取失败，可能原因包括：

- 基金代码不存在。
- AkShare 上游接口暂时不可用。
- 网络连接不稳定。
- 基金成立时间太短，起始日期之后的数据不足。

处理方式：

- 换几只常见基金代码测试。
- 把起始日期调晚一些。
- 使用默认基金池演示。
